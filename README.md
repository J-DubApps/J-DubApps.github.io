Julianwest.me static content was originally migrated to Jekyll, then on to Hugo.  Read more about those Static Site Generators here: https://gohugo.io / https://jekyllrb.com

Hugo is elegant "CMS" that focuses on text and static markdown generated pages site-wide, rather than using a database and dynamically-generated individual pages.  It's well-worth a look.

A word on the old stuff I used -- Blogger, Squarespace, and Ghost

I used these platforms during the lifespan of julianwest.me and the now-defunct mydigitalpathos.com, and I loved many things about each platofrm.  I think those solutions were great for the time, especially for non-coders.  Overall I could never use WordPress or Movabletype, just less flexible though the whole web for a time seemed like it was WordPress. Managing templates is annoying.  Gimme plain text and Markdown any day.  A bit less to maintain, design-wise, and more opportunity to just write or code without living in js template hell with endless CSS tweaking.
